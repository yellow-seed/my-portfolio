# Claude-Inspired Design System

This design system is inspired by Claude's warm and welcoming brand identity, featuring a carefully crafted color palette that emphasizes approachability and clarity.

## Philosophy

The color palette is built around three core principles:

1. **Warmth & Approachability**: Using orange and coral tones to create an inviting experience
2. **Clarity & Hierarchy**: Neutral tones that provide clear visual hierarchy
3. **Accessibility**: All color combinations meet WCAG 2.1 AA standards

## Usage Guidelines

### Tailwind CSS Classes

All colors are available as Tailwind utility classes:

```tsx
// Backgrounds
<div className="bg-primary-500">Primary background</div>
<div className="bg-neutral-100">Light background</div>

// Text
<h1 className="text-neutral-900">Heading</h1>
<p className="text-neutral-600">Body text</p>

// Borders
<div className="border border-neutral-200">Card</div>
<div className="border-2 border-primary-500">Highlighted</div>

// Hover states
<button className="bg-primary-500 hover:bg-primary-600">
  Button
</button>
```

### CSS Variables

For more flexibility, use CSS variables:

```css
.custom-element {
  background-color: var(--color-primary-500);
  color: var(--color-text-primary);
  border: 1px solid var(--color-border-default);
}
```

### TypeScript

Import colors programmatically:

```typescript
import { colors, semanticColors } from '@/design-system/colors';

const brandColor = colors.primary[500]; // "#F97316"
const textColor = semanticColors.text; // "#1C1917"
```

## Best Practices

### Do ✅

- Use primary-500 for main call-to-action buttons
- Use neutral scale for text hierarchy (900 for headings, 600 for body)
- Apply hover states by darkening colors 1-2 shades
- Use semantic colors (success/warning/error) for status indicators

### Don't ❌

- Don't use bright colors for large background areas
- Don't mix adjacent color shades in the same component
- Don't use primary color directly for body text
- Don't forget to test color contrast for accessibility
